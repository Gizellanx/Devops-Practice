# Linux Practice

Hands-on practice with Linux commands and real-world troubleshooting scenarios.

## Objective

To build practical experience using Linux for log analysis, system navigation, and troubleshooting tasks used in DevOps.

## What I'm Practising

- Navigating the Linux file system
- Working with files and directories
- Analysing log files
- Using command-line tools (awk, sort, uniq)
- Understanding pipes and command chaining
- Extracting and processing data from logs

## Exercises

### Day 1 – Navigation & File Management
- Practised basic Linux commands
- Learned how to move through directories and manage files

### Day 2 – Log Troubleshooting
- Worked with log files to identify system issues
- Practised reading and understanding log outputs

### Day 3 – Log File Analysis (Most Active IP)
- Analysed a web server log file
- Identified the IP address with the highest number of requests
- Used awk, sort, uniq, and pipes to process data
- Saved output to a file using redirection (>)

### Day 4 – Linux Log Investigation Refresh
- Refreshed Linux log investigation workflows
- Practised grep, tail, and cat commands using access logs
- Revisited terminal-based troubleshooting and log filtering

## Skills Gained

- Log analysis using Linux CLI
- Data extraction from files
- Command chaining using pipes (|)
- Understanding how logs reflect system activity

## Why This Matters

Linux is widely used in DevOps environments. These exercises simulate real tasks such as analysing logs, troubleshooting systems, and understanding system behaviour.
