# Python (DevOps Focus)

This folder contains Python exercises and mini-projects focused on real-world DevOps tasks such as log analysis, monitoring, automation, and working with structured data.

## Focus

- Automating repetitive tasks
- Processing and analysing log files
- Extracting useful information from data
- Building simple monitoring and detection tools
- Understanding how Python supports DevOps workflows

## Exercises

### Day 2 – Log File Analyser (System Logs)

- Read a log file line by line
- Detected errors and warnings
- Filtered CPU, disk, and memory-related issues
- Counted and summarised results

### Day 5 – Web Log Analysis (Access Logs)

- Extracted IP addresses from web server logs
- Identified the most active IP address
- Detected failed requests (4xx and 5xx status codes)
- Flagged suspicious IP addresses using thresholds

### Day 6 – Python Refresh

- Reviewed previous log analysis exercises
- Revisited conditions, loops, and file handling
- Refreshed monitoring workflows and troubleshooting logic

### Day 7 – System Monitoring

- Used Python to monitor CPU, memory, and disk usage
- Introduced basic monitoring alerts using conditions
- Connected Linux monitoring concepts with Python automation

### Day 8 – JSON Processing

- Learned how JSON stores structured information
- Read and processed JSON data using Python
- Extracted values from monitoring-related data
- Used conditions to analyse structured information

## Skills Practised

- File handling
- Loops and iteration
- Conditional logic
- String processing
- Data filtering
- Log analysis
- Monitoring concepts
- JSON processing
- Basic automation

## Key Learning

One of the biggest lessons so far has been seeing how the same Python fundamentals can be applied to different problems.

Whether analysing logs, monitoring systems, detecting suspicious activity, or processing JSON data, the overall structure remains similar while the logic changes depending on the task.

This reflects real DevOps work, where scripts are often adapted and reused for different monitoring, troubleshooting, and automation scenarios.
